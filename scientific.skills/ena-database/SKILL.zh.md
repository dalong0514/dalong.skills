# ENA 数据库

## 概述

欧洲核苷酸档案库 (ENA) 是核苷酸序列数据和相关元数据的综合公共存储库。通过基因组学和生物信息学管道的 REST API 和 FTP 访问和查询 DNA/RNA 序列、原始读数、基因组组装和功能注释。

## 何时使用此技能

该技能应该在以下情况下使用：

- 通过登录检索核苷酸序列或原始测序读数
- 按元数据标准搜索样本、研究或组件
- 下载 FASTQ 文件或基因组组件进行分析
- 查询生物的分类信息
- 访问序列注释和功能数据
- 将 ENA 数据整合到生物信息学管道中
- 对相关数据库进行交叉引用搜索
- 通过 FTP 或 Aspera 批量下载数据集

## 核心能力

### 1. 数据类型和结构

ENA 将数据组织为分层对象类型：

**研究/项目** - 分组相关数据并控制发布日期。研究是引用存档数据的主要单位。

**样品** - 代表产生测序文库的生物材料单位。在提交大多数数据类型之前必须注册样本。

**原始读取** - 包括：
- **实验**：有关测序方法、文库制备和仪器详细信息的元数据
- **运行**：对包含来自单个测序运行的原始测序读数的数据文件的引用

**程序集** - 不同完成水平的基因组、转录组、宏基因组或元转录组程序集。

**序列** - 存储在 EMBL 核苷酸序列数据库中的组装和注释序列，包括编码/非编码区域和功能注释。

**分析** - 序列数据计算分析的结果。

**分类记录** - 分类信息，包括谱系和等级。

### 2. 编程访问

ENA 提供多个 REST API 用于数据访问。请参阅 `references/api_reference.md` 以获取详细的端点文档。

**关键API：**

**ENA Portal API** - 跨所有 ENA 数据类型的高级搜索功能
- 文档：https://www.ebi.ac.uk/ena/portal/api/doc
- 用于复杂查询和元数据搜索

**ENA 浏览器 API** - 直接检索记录和元数据
- 文档：https://www.ebi.ac.uk/ena/browser/api/doc
- 用于按加入下载特定记录
- 返回XML格式的数据

**ENA Taxonomy REST API** - 查询分类信息
- 访问谱系、等级和相关分类数据

**ENA 交叉参考服务** - 从外部数据库访问相关记录
- 端点：https://www.ebi.ac.uk/ena/xref/rest/

**CRAM 参考注册表** - 检索参考序列
- 端点：https://www.ebi.ac.uk/ena/cram/
- 通过MD5或SHA1校验和查询

**速率限制**：所有 API 的速率限制为每秒 50 个请求。超过此值将返回 HTTP 429（请求过多）。

### 3. 搜索和检索数据

**基于浏览器的搜索：**
- 跨所有领域的免费文本搜索
- 序列相似性搜索（BLAST集成）
- 交叉引用搜索以查找相关记录
- 使用规则空间查询生成器进行高级搜索

**程序化查询：**
- 使用 Portal API 进行大规模高级搜索
- 按数据类型、日期范围、分类或元数据字段过滤
- 将结果下载为表格元数据摘要或 XML 记录

**API 查询模式示例：**
```python
import requests

# Search for samples from a specific study
base_url = "https://www.ebi.ac.uk/ena/portal/api/search"
params = {
    "result": "sample",
    "query": "study_accession=PRJEB1234",
    "format": "json",
    "limit": 100
}

response = requests.get(base_url, params=params)
samples = response.json()
```

### 4. 数据检索格式

**元数据格式：**
- XML（本机 ENA 格式）
- JSON（通过门户 API）
- TSV/CSV（表格摘要）

**序列数据：**
- FASTQ（原始读取）
- BAM/CRAM（对齐读取）
- FASTA（组装序列）
- EMBL 平面文件格式（带注释的序列）

**下载方法：**
- 直接API下载（小文件）
- 用于批量数据传输的FTP
- Aspera 用于高速传输大型数据集
- 用于批量下载的 enaBrowserTools 命令行实用程序

### 6. 与分析管道集成

**批量下载模式：**
1. 使用 Portal API 搜索符合条件的种质
2. 从搜索结果中提取文件 URL
3.通过FTP或使用enaBrowserTools下载文件
4. 在管道中处理下载的数据

**BLAST 集成：**
与 EBI 的 NCBI BLAST 服务 (REST/SOAP API) 集成，以针对 ENA 序列进行序列相似性搜索。

### 7. 最佳实践

**速率限制：**
- 接收 HTTP 429 响应时实施指数退避
- 批量请求尽可能保持在 50 请求/秒限制内
- 对大型数据集使用批量下载工具，而不是迭代 API 调用

**数据引用：**
- 发表时始终引用研究/项目收录
- 包括所使用的特定样品、运行或组件的登录号

**API 响应处理：**
- 在处理响应之前检查 HTTP 状态代码
- 使用适当的 XML 库（不是正则表达式）解析 XML 响应
- 处理大型结果集的分页

**性能：**
- 使用 FTP/Aspera 下载大文件 (>100MB)
- 当仅需要元数据时，优先选择 TSV/JSON 格式而不是 XML
- 处理许多记录时在本地缓存分类查找
