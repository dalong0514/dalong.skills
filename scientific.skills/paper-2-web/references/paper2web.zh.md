# Paper2Web：学术主页生成

## 概述

Paper2Web 将学术论文转换为交互式、可探索的学术主页。与传统方法（直接生成、基于模板或 HTML 转换）不同，Paper2Web 通过迭代细化过程创建布局感知的交互式网站。

## 核心能力

### 1. 布局感知生成
- 分析论文结构和内容组织
- 创建响应式、多部分布局
- 根据论文类型（研究文章、评论、预印本等）调整设计

### 2.互动元素
- 详细内容的可扩展部分
- 交互式图表
- 嵌入引文和参考文献
- 导航菜单，方便浏览
- 移动响应式设计

### 3.内容细化
该系统使用迭代管道：
1. 初始内容提取和结构化
2. 具有视觉层次结构的布局生成
3、交互元素整合
4.审美精致
5. 质量评估和验证

## 用法

### 基本网站生成

```bash
python pipeline_all.py \
  --input-dir "path/to/papers" \
  --output-dir "path/to/output" \
  --model-choice 1
```

### 参数

- `--input-dir`：包含纸质文件（PDF 或 LaTeX）的目录
- `--output-dir`：生成的网站文件的目录
- `--model-choice`：LLM模型选择（1=GPT-4，2=GPT-4.1）
- `--enable-logo-search`：使用 Google 搜索 API 查找机构徽标（可选）

### 输入格式要求

**支持的输入格式：**
1. **LaTeX 源**（最佳结果首选）
   - 主文件：`main.tex`
   - 包括所有引用的图形、表格和参考书目文件
   - 将每篇论文组织在一个目录中

2. **PDF文件**
   - 带有可选文本的高质量 PDF
   - 嵌入的图形应该是高分辨率的
   - 正确的节标题和结构

## 最佳实践

### 输入准备
1. **尽可能使用 LaTeX**：提供最佳的结构提取
2. **包括所有资产**：图形、表格和参考书目文件
3. **干净格式**：删除编译工件和临时文件
4. **高质量图形**：如果可用，请使用矢量格式（PDF、SVG）

### 型号选择
- **GPT-4**：质量和成本的最佳平衡
- **GPT-4.1**：最新功能，成本较高
- **GPT-3.5-turbo**：处理速度更快，适合简单论文

### 输出优化
1. 检查生成内容的准确性
2. 检查所有图形是否正确渲染
3. 测试交互元素功能
4. 验证移动响应能力
5. 验证外部链接

## 限制

- 复杂的数学方程可能需要人工审核
- PDF 中的多列布局可能会影响提取质量
- 大论文（>50 页）可能需要延长处理时间
- 一些特殊的图形类型可能需要手动调整
