# Paper2All：学术论文转换管道

## 概述

这项技能可以使用 Paper2All 自主管道将学术论文转换为多种推广和演示格式。该系统将研究论文（LaTeX 或 PDF）转换为三个主要输出：

1. **Paper2Web**：具有布局感知设计的交互式、可探索的学术主页
2. **Paper2Video**：带有旁白、幻灯片和可选的主讲人的专业演示视频
3. **Paper2Poster**：具有专业布局的可打印会议海报

该管道使用法学硕士支持的内容提取、设计生成和迭代细化来创建适合会议、期刊、预印本存储库和学术推广的高质量输出。

## 何时使用此技能

在以下情况下使用此技能：

- **创建会议材料**：学术会议的海报、演示视频和配套网站
- **促进研究**：将已发表的论文或预印本转换为易于访问、引人入胜的网络格式
- **准备演示文稿**：从纸质内容生成视频摘要或完整演示视频
- **传播发现**：为社交媒体、实验室网站或机构展示制作宣传材料
- **增强预印本**：向 bioRxiv、arXiv 或其他预印本提交内容添加交互式主页
- **批量处理**：同时生成多篇论文的宣传材料

**主要特点**：

- 审美细化和质量评估

## 输入要求

### 支持的输入格式

**1. LaTeX 源**（推荐）：
```
paper_directory/
├── main.tex              # Main paper file
├── sections/             # Optional: split sections
├── figures/              # All figure files
├── tables/               # Table files
└── bibliography.bib      # References
```

**2. PDF**：
- 带有嵌入字体的高质量 PDF
- 可选文本（非扫描图像）
- 高分辨率图形（300+ DPI 优先）

### 输入组织

**单张纸**：
```bash
input/
└── paper_name/
    ├── main.tex (or paper.pdf)
    ├── figures/
    └── bibliography.bib
```

**多篇论文**（批量处理）：
```bash
input/
├── paper1/
│   └── main.tex
├── paper2/
│   └── main.tex
└── paper3/
    └── main.tex
```

## 常用参数

### 型号选择
- `--model-choice 1`：GPT-4（质量和成本的最佳平衡）
- `--model-choice 2`：GPT-4.1（最新功能，成本较高）
- `--model_name_t gpt-3.5-turbo`：更快、成本更低（质量可接受）

### 组件选择
- `--generate-website`：启用网站生成
- `--generate-poster`：启用海报生成
- `--generate-video`：启用视频生成
- `--enable-talking-head`：向视频添加头部说话（需要 GPU）

### 定制
- `--poster-width-inches [width]`：自定义海报宽度
- `--poster-height-inches [height]`：自定义海报高度
- `--video-duration [seconds]`：目标视频长度
- `--enable-logo-search`：自动机构徽标发现

## 最佳实践

### 输入准备
1. **尽可能使用 LaTeX**：提供最佳的内容提取和结构
2. **正确组织文件**：将所有资产（图形、表格、参考书目）保留在论文目录中
3. **高质量图形**：使用矢量格式（PDF、SVG）或高分辨率光栅（300+ DPI）
4. **Clean LaTeX**：删除编译工件，确保源代码编译成功

### 模型选择策略
- **GPT-4**：最适合生产质量的输出、会议、出版物
- **GPT-4.1**：当您需要最新功能或最佳质量时使用
- **GPT-3.5-turbo**：用于快速草稿、测试或简单论文

### 组件优先级
对于紧迫的期限，请按以下顺序生成：
1. **网站**（最快、最通用，约 15-30 分钟）
2. **海报**（中等速度，打印截止日期，约10-20分钟）
3. **视频**（最慢，可以稍后生成，约20-60分钟）

### 质量保证
在最终确定输出之前：
1. **网站**：在多个设备上进行测试，验证所有链接是否有效，检查图形质量
2. **海报**：打印测试页，验证 3-6 英尺的文本可读性，检查颜色
3. **视频**：观看整个视频，验证音频同步，在不同设备上测试

## 故障排除

### 常见问题

**LaTeX 解析错误**：
- 确保 LaTeX 源代码编译成功：`pdflatex main.tex`
- 检查所有引用的文件是否存在
- 验证没有自定义包阻止解析

**人物质量较差**：
- 使用矢量格式（PDF、SVG、EPS）代替光栅
- 确保光栅图像的 DPI 超过 300
- 检查编译后的 PDF 中的图形是否正确呈现

**视频生成失败**：
- 验证足够的磁盘空间（建议 5GB+）
- 检查所有已安装的依赖项（LibreOffice、Poppler）
- 查看输出目录中的错误日志

**海报布局问题**：
- 验证海报尺寸是否合理（24英寸-72英寸范围）
- 检查内容长度（非常长的论文可能需要手动整理）
- 确保人物具有适合海报尺寸的分辨率

**API错误**：
- 验证 `.env` 文件中的 API 密钥
- 检查API积分余额
- 确保没有速率限制（等待并重试）
