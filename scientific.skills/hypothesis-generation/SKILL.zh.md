# 科学假设的生成

## 概述

假设生成是开发可检验解释的系统过程。根据观察形成基于证据的假设、设计实验、探索相互竞争的解释并做出预测。将这项技能应用于跨领域的科学探究。

## 何时使用此技能

该技能应该在以下情况下使用：
- 根据观察或初步数据提出假设
- 设计实验来测试科学问题
- 探索对现象的相互竞争的解释
- 制定可测试的研究预测
- 进行基于文献的假设生成
- 规划跨科学领域的机制研究

## 工作流程

遵循这个系统过程来产生可靠的科学假设：

### 1.了解现象

首先澄清需要解释的观察结果、问题或现象：

- 确定需要解释的核心观察或模式
- 定义现象的范围和边界
- 注意任何限制或特定上下文
- 澄清已知的与不确定的
- 确定相关的科学领域

### 2. 进行全面的文献检索

搜索现有科学文献，根据当前证据提出假设。使用 PubMed（针对生物医学主题）和一般网络搜索（针对更广泛的科学领域）：

**对于生物医学主题：**
- 使用带有 PubMed URL 的 WebFetch 来访问相关文献
- 搜索最近的评论、荟萃分析和初步研究
- 寻找相似的现象、相关的机制或类似的系统

**对于所有科学领域：**
- 使用网络搜索查找最近的论文、预印本和评论
- 寻找既定的理论、机制或框架
- 找出当前理解中的差距

**搜索策略：**
- 从广泛的搜索开始，了解情况
- 缩小到特定的机制、途径或理论
- 寻找矛盾的发现或未解决的争论
- 请参阅 `references/literature_search_strategies.md` 了解详细的搜索技巧

### 3. 综合现有证据

分析并整合文献检索的结果：

- 总结目前对该现象的理解
- 确定可能适用的既定机制或理论
- 注意相互矛盾的证据或替代观点
- 认识差距、限制或未解答的问题
- 识别相关系统或领域的类比

### 4. 生成竞争假设

提出 3-5 个可以解释该现象的不同假设。每个假设应该：

- 提供机械解释（不仅仅是描述）
- 与其他假设区分开来
- 从文献综合中汲取证据
- 考虑不同层次的解释（分子、细胞、系统、群体等）

**生成假设的策略：**
- 应用类似系统中的已知机制
- 考虑多种致病途径
- 探索不同尺度的解释
- 对现有解释中的假设提出质疑
- 以新颖的方式组合机制

### 5. 评估假设质量

根据 `references/hypothesis_quality_criteria.md` 中既定的质量标准评估每个假设：

**可检验性：** 假设可以通过实证检验吗？
**可证伪性：**什么观察结果可以反驳它？
**简约：** 这是符合证据的最简单的解释吗？
**解释力：** 它解释了多少现象？
**范围：** 它涵盖哪些观察范围？
**一致性：** 是否符合既定原则？
**新颖性：**它是否提供了超出现有解释的新见解？

明确记下每个假设的优点和缺点。

### 6. 设计实验测试

对于每个可行的假设，提出具体的实验或研究来检验它。请参阅 `references/experimental_design_patterns.md` 了解常见方法：

**实验设计元素：**
- 将测量或观察什么？
- 需要哪些比较或控制？
- 将使用什么方法或技术？
- 什么样的样本量或统计方法是合适的？
- 潜在的混淆是什么以及如何解决它们？

**考虑多种方法：**
- 实验室实验（体外、体内、计算）
- 观察性研究（横断面、纵向、病例对照）
- 临床试验（如果适用）
- 自然实验或准实验设计

### 7. 制定可测试的预测

对于每个假设，生成具体的定量预测：

- 说明如果假设正确，应该观察到什么
- 尽可能指定预期影响的方向和程度
- 确定预测成立的条件
- 区分相互竞争的假设之间的预测
- 注意会证伪假设的预测

### 8.呈现结构化输出

使用 `assets/hypothesis_output_template.md` 中的模板以清晰、一致的格式提出假设：

**标准结构：**
1. **背景和背景** - 现象和文献总结
2. **竞争假设** - 带有机械解释的列举假设
3. **质量评估** - 每个假设的评估
4. **实验设计** - 针对每个假设的建议测试
5. **可测试的预测** - 具体的、可测量的预测
6. **关键比较** - 如何区分假设

## 质量标准

确保所有生成的假设符合这些标准：

- **基于证据：** 以现有文献为基础并引用
- **可测试：** 包括具体的、可测量的预测
- **机制：**解释如何/为什么，而不仅仅是什么
- **全面：** 考虑替代解释
- **严格：** 包括实验设计来测试预测
