# 专利审查数据系统 (PEDS) API 参考

## 概述

专利审查数据系统 (PEDS) 提供对 USPTO 专利申请和提交状态记录的访问。它包含书目数据、已发表的文档信息和专利期限扩展数据。

**数据覆盖范围：** 1981 年至今（部分数据可追溯到 1935 年）

**基本 URL：** 通过 USPTO 开放数据门户访问

## PEDS 提供什么

PEDS 提供专利申请的全面交易历史记录和状态信息：

- **书目数据** - 申请号、申请日期、标题、发明人、受让人
- **已发表的文件** - 出版号和日期
- **交易历史记录** - 所有检查事件以及日期、代码和描述
- **专利期限调整** - PTA/PTE 信息
- **申请状态** - 当前状态和状态代码
- **文件包装访问** - 起诉文件的链接

## 主要特点

1. **交易活动** - 完整的检查时间表，包括交易日期、代码和描述
2. **状态信息** - 当前应用程序状态和状态代码
3. **书目更新** - 发明人、受让人、标题随时间的变化
4. **家庭数据** - 相关应用程序和连续性数据
5. **审查意见通知跟踪** - 邮寄日期和审查意见通知信息

## 数据字段

### 书目字段

- `application_number` - 申请号
- `filing_date` - 提交日期
- `patent_number` - 专利号（如果已授予）
- `patent_issue_date` - 颁发日期（如果授予）
- `title` - 申请/专利标题
- `inventors` - 发明人列表
- `assignees` - 受让人列表
- `app_type` - 应用程序类型（实用、设计、工厂、重新发布）
- `app_status` - 当前申请状态
- `app_status_date` - 状态日期

### 交易字段

- `transaction_date` - 交易日期
- `transaction_code` - USPTO 事件代码
- `transaction_description` - 事件描述
- `mail_date` - 邮件室日期（用于办公室行动）

### 专利术语数据

- `pta_pte_summary` - 专利期限调整/延期摘要
- `pta_pte_history` - 术语计算的历史记录

## 状态代码

常见的应用程序状态代码：

- **专利案例** - 已获得专利
- **放弃** - 申请被放弃
- **待处理** - 申请正在审查中
- **允许** - 申请已被允许，等待发布
- **最终拒绝** - 发出最终拒绝
- **非最终拒绝** - 发出非最终拒绝
- **已提交回复** - 已提交申请人回复

## 交易代码

常见的交易代码包括：

- **CTNF** - 邮寄非最终拒绝邮件
- **CTFR** - 最终拒绝邮寄
- **AOPF** - 邮寄办公室行动
- **书面** - 已提交回复
- **NOA** - 邮寄津贴通知
- **ISS.FEE** - 发行费用支付
- **ABND** - 申请被放弃

OCE 专利审查状态/事件代码 API 中提供完整的代码列表。

## 与其他 API 集成

PEDS 数据可以与其他 USPTO API 结合使用：

- **审查意见通知文本 API** - 使用申请号检索审查意见通知全文
- **专利转让搜索** - 查找所有权变更
- **PTAB API** - 检查上诉程序

## 重要提示

1. **PAIR Bulk Data (PBD) 已停用** - 使用 PEDS 代替
2. **数据更新** - PEDS定期更新，但可能有1-2天的滞后
3. **申请号** - 使用标准化格式（无斜杠或空格）
4. **连续性数据** - 在事务历史记录中跟踪的父/子应用程序

## 最佳实践

1. **批量请求** - 对多个应用程序使用批量检索
2. **缓存数据** - 避免同一应用程序的冗余 API 调用
3. **监控更新** - 定期检查交易更新
4. **处理缺失数据** - 并非所有应用程序都填充所有字段
5. **解析交易代码** - 使用代码描述方便用户展示
