# 实验设计清单

## 研究问题的提出

### 问题格式正确吗？
- [ ] **具体：** 明确定义的变量和关系
- [ ] **可回答：** 可以用可用的方法解决
- [ ] **相关：** 解决知识或实际需求方面的差距
- [ ] **可行：** 资源、时间和道德考虑允许
- [ ] **可证伪：** 如果不正确，可以证明是错误的

### 你复习过文献吗？
- [ ] 确定已知的内容
- [ ] 发现需要解决的差距或矛盾
- [ ] 从方法论的成功和失败中学习
- [ ] 确定适当的成果衡量标准
- [ ] 确定该领域的典型效应大小

## 假设发展

### 你的假设可以检验吗？
- [ ] 做出具体的、可量化的预测
- [ ] 变量是可操作定义的
- [ ] 指定预期关系的方向/性质
- [ ] 可以通过潜在的观察结果来证伪

### 假设的类型
- [ ] **原假设 (H₀):** 不存在影响/关系
- [ ] **替代假设 (H₁):** 效应/关系存在
- [ ] **定向与非定向：** 单尾与双尾测试

## 研究设计选择

### 什么类型的学习是合适的？

**实验（干预）研究：**
- [ ] **随机对照试验 (RCT)：** 因果关系的黄金标准
- [ ] **准实验：** 非随机分配但操纵
- [ ] **受试者内：** 所有条件下的相同参与者
- [ ] **受试者之间：** 每个条件的不同参与者
- [ ] **阶乘：** 多个自变量
- [ ] **交叉：** 参与者按顺序接受多次干预

**观察研究：**
- [ ] **群组：** 随着时间的推移关注群组
- [ ] **病例对照：** 比较有/无结果的病例
- [ ] **横截面：** 某一时间点的快照
- [ ] **生态：** 人口层面的数据

**考虑：**
- [ ] 可以随机分配参与者吗？
- [ ] 你能操纵自变量吗？
- [ ] 结果是罕见（有利于病例对照）还是常见？
- [ ] 是否需要建立时间顺序？
- [ ] 考虑到道德和实际限制，什么是可行的？

## 变量

### 自变量（操纵/预测变量）
- [ ] 明确定义并可操作
- [ ] 选择适当的级别/类别
- [ ] 操纵足以检验假设
- [ ] 计划进行操作检查（如果适用）

### 因变量（结果/响应）
- [ ] 直接测量兴趣结构
- [ ] 经过验证且可靠的测量
- [ ] 足够灵敏，可以检测预期效果
- [ ] 适用于计划的统计分析
- [ ] 明确指定主要结果

### 控制变量
- [ ] **确定的混杂变量：**
  - 同时影响 IV 和 DV 的变量
  - 对调查结果的替代解释
- [ ] **控制策略：**
  - 随机化
  - 匹配
  - 分层
  - 统计调整
  - 限制（纳入/排除标准）
  - 致盲

### 无关变量
- [ ] 已识别的潜在噪声源
- [ ] 标准化程序，最大限度地减少
- [ ] 控制环境因素
- [ ] 时间、设置、设备标准化

## 采样

### 人口定义
- [ ] **目标人群：** 您想要推广到的人
- [ ] **可接触人群：** 您实际上可以从谁那里进行抽样
- [ ] **示例：** 实际参与人员
- [ ] 这些记录之间的差异

### 抽样方法
- [ ] **概率抽样（优先考虑普遍性）：**
  - 简单随机抽样
  - 分层抽样
  - 整群抽样
  - 系统抽样
- [ ] **非概率抽样（常见但限制普遍性）：**
  - 方便取样
  - 有目的的抽样
  - 雪球采样
  - 配额抽样

### 样本量
- [ ] **进行先验功效分析**
  - 预期效果大小（来自文献或试点）
  - 所需功率（通常为 0.80 或 0.90）
  - 显着性水平（通常为 0.05）
  - 使用统计检验
- [ ] 考虑预期的流失/退出
- [ ] 足以进行计划的亚组分析
- [ ] 承认实际限制

### 纳入/排除标准
- [ ] 定义明确且合理
- [ ] 不过度限制（限制普遍性）
- [ ] 基于理论或实践考虑
- [ ] 解决道德问题
- [ ] 一致地记录和应用

## 盲法和随机化
### 随机化
- [ ] **随机化的内容：**
  - 参与者分配条件
  - 条件顺序（科目内）
  - 提供的刺激/物品
- [ ] **随机化方法：**
  - 计算机生成的随机数
  - 随机数字表
  - 抛硬币（适用于非常小的研究）
- [ ] **分配隐藏：**
  - 招募前生成的序列
  - 分配在注册后隐藏
  - 按顺序编号的密封信封（如果需要）
- [ ] **分层随机化：**
  - 平衡各组之间的重要变量
  - 分组随机化以确保组大小相等
- [ ] **检查随机化：**
  - 在基线上比较各组
  - 报告任何显着差异

### 致盲
- [ ] **单盲：** 参与者不知道小组分配
- [ ] **双盲：** 参与者和研究人员不知道
- [ ] **三盲：** 参与者、研究人员和数据分析师不知道
- [ ] **致盲可行性：**
  - 真正的致盲可能吗？
  - 需要安慰剂/假对照吗？
  - 干预措施的外观相同？
- [ ] **盲法检查：**
  - 评估是否维持致盲
  - 要求参与者/研究人员猜测作业

## 对照组和条件

### 什么类型的控制？
- [ ] **无治疗对照：** 病情的自然病程
- [ ] **安慰剂对照：** 用于比较的惰性处理
- [ ] **主动对照：** 标准治疗比较
- [ ] **等待名单控制：** 延迟治疗
- [ ] **注意力控制：** 匹配无活性成分的接触时间

### 多个条件
- [ ] 多因素因子设计
- [ ] 剂量反应关系评估
- [ ] 通过成分分析进行机制测试

## 程序

### 协议开发
- [ ] **详细的书面协议：**
  - 分步程序
  - 标准化指令脚本
  - 处理问题的决策规则
  - 数据收集表格
- [ ] 在主要研究之前进行试点测试
- [ ] 员工接受标准培训
- [ ] 计划进行合规性监控

### 标准化
- [ ] 对所有参与者的相同说明
- [ ] 相同的设备和材料
- [ ] 尽可能相同的环境/设置
- [ ] 相同的评估时间
- [ ] 记录的协议偏差

### 数据收集
- [ ] **收集时：**
  - 基线测量
  - 干预后
  - 后续时间点
- [ ] **谁收集：**
  - 训练有素的研究人员
  - 尽可能设盲
  - 建立评估者间的可靠性
- [ ] **如何收集：**
  - 有效、可靠的仪器
  - 规范化管理
  - 如果可能的话多种方法（三角测量）

## 测量

### 有效性
- [ ] **表面有效性：** 似乎可以测量构造
- [ ] **内容有效性：** 涵盖构造的所有方面
- [ ] **标准有效性：** 与黄金标准相关
  - 并发有效性
  - 预测有效性
- [ ] **构造有效性：** 测量理论构造
  - 收敛效度（与相关措施相关）
  - 区分有效性（与不相关的措施不相关）

### 可靠性
- [ ] **测试再测试：** 随着时间的推移保持一致
- [ ] **内部一致性：** 项目测量相同的结构（Cronbach's α）
- [ ] **评估者间的可靠性：** 评估者之间的协议（Cohen's κ、ICC）
- [ ] **并行形式：** 替代版本一致

### 测量注意事项
- [ ] 尽可能首选客观措施
- [ ] 可用时使用经过验证的仪器
- [ ] 关键结构的多重测量
- [ ] 考虑变化的敏感性
- [ ] 避免地板/天花板效应
- [ ] 适当的响应格式
- [ ] 适当的召回期限
- [ ] 考虑文化适宜性

## 偏差最小化

### 选择偏差
- [ ] 尽可能随机抽样
- [ ] 明确定义的资格标准
- [ ] 记录谁拒绝以及原因
- [ ] 最小化自我选择

### 性能偏差
- [ ] 标准化协议
- [ ] 提供者致盲
- [ ] 监控协议遵守情况
- [ ] 文档偏差

### 检测偏差
- [ ] 结果评估者采用盲法
- [ ] 尽可能采取客观措施
- [ ] 标准化评估程序
- [ ] 多个评估者进行可靠性检查

### 损耗偏差
- [ ] 最小化辍学的策略
- [ ] 追踪辍学原因
- [ ] 比较辍学者和完成者
- [ ] 计划进行意向治疗分析

### 报告偏见
- [ ] 预登记学习分析计划
- [ ] 指定主要结果和次要结果
- [ ] 承诺报告所有结果
- [ ] 区分计划分析和探索性分析

## 数据管理

### 数据收集
- [ ] 数据收集表格的设计和测试
- [ ] REDCap、Qualtrics 或类似平台
- [ ] 范围检查和验证规则
- [ ] 定期备份
- [ ] 安全存储（如果需要，符合 HIPAA/GDPR）

### 数据质量
- [ ] 实时数据验证
- [ ] 定期质量检查
- [ ] 监控缺失数据模式
- [ ] 异常值的识别和调查
- [ ] 记录协议偏差

### 数据安全
- [ ] 去识别化程序
- [ ] 访问控制
- [ ] 审计追踪
- [ ] 遵守法规（IRB、HIPAA、GDPR）

## 统计分析规划

### 分析计划（数据收集前预先指定）
- [ ] **初步分析：**
  - 指定的统计测试
  - 明确陈述假设
  - 显着性水平设置（通常 α = .05）
  - 一尾或二尾
- [ ] **二次分析：**
  - 明确指定为次要
  - 如此标记的探索性分析
- [ ] **多重比较：**
  - 指定调整方法（如果需要）
  - 主要成果可防止通货膨胀

### 假设
- [ ] 确定的统计测试假设
- [ ] 计划检查假设
- [ ] 备份非参数替代方案
- [ ] 考虑的转换选项

### 缺失数据
- [ ] 预计缺失数量
- [ ] 缺失数据机制（MCAR、MAR、MNAR）
- [ ] 处理策略：
  - 完整的案例分析
  - 多重插补
  - 最大似然
- [ ] 计划进行敏感性分析

### 效果大小
- [ ] 确定适当的效应量测量
- [ ] 将与 p 值一起报告
- [ ] 计划的置信区间

### 统计软件
- [ ] 所选软件（R、SPSS、Stata、Python 等）
- [ ] 记录版本
- [ ] 预先准备的分析脚本
- [ ] 将提供（开放科学）

## 道德考虑

### 道德批准
- [ ] 获得 IRB/道德委员会批准
- [ ] 研究注册（ClinicalTrials.gov 等）（如果适用）
- [ ] 协议遵循赫尔辛基宣言或同等文件

### 知情同意书
- [ ] 自愿参与
- [ ] 易于理解的解释
- [ ] 披露的风险和收益
- [ ] 退出而不受处罚的权利
- [ ] 隐私保护解释
- [ ] 披露补偿

### 风险收益分析
- [ ] 潜在收益大于风险
- [ ] 风险最小化
- [ ] 弱势群体受到保护
- [ ] 数据安全监控（如果高风险）

### 保密
- [ ] 数据已去识别化
- [ ] 安全存储
- [ ] 访问受限
- [ ] 举报不允许重新识别

## 有效性威胁

### 内部有效性（因果关系）
- [ ] **历史记录：** 测量之间的外部事件
- [ ] **成熟度：** 参与者随时间的变化
- [ ] **测试：** 重复测量的效果
- [ ] **仪器：** 测量值随时间的变化
- [ ] **回归均值：** 极端分数变得不那么极端
- [ ] **选择：** 组的基线不同
- [ ] **损耗：** 差异辍学
- [ ] **扩散：** 对照组接受治疗元素

### 外部有效性（普遍性）
- [ ] 总体样本代表
- [ ] 设置真实/自然
- [ ] 现实世界实施的典型处理
- [ ] 结果措施在生态上有效
- [ ] 适当的时间范围

### 构造有效性（测量）
- [ ] 措施实际上利用了预期的结构
- [ ] 操作符合理论定义
- [ ] 没有混淆结构
- [ ] 构造的充分覆盖

### 统计结论的有效性
- [ ] 足够的统计功效
- [ ] 满足假设
- [ ] 使用适当的测试
- [ ] Alpha 级别适当
- [ ] 解决多重比较问题

## 报告和透明度

### 预注册
- [ ] 研究预先注册（OSF、ClinicalTrials.gov、AsPredicted）
- [ ] 先验假设
- [ ] 记录分析计划
- [ ] 区分确认性和探索性

### 报告指南
- [ ] **RCT：** CONSORT 检查表
- [ ] **观察性研究：** STROBE 检查表
- [ ] **系统审查：** PRISMA 检查表
- [ ] **诊断研究：** STARD 检查表
- [ ] **定性研究：** COREQ 检查表
- [ ] **病例报告：** CARE 指南

### 透明度
- [ ] 报告的所有措施
- [ ] 披露所有操纵行为
- [ ] 样本量确定说明
- [ ] 排除标准和报告的数量
- [ ] 记录有人员流失
- [ ] 注意到与协议的偏差
- [ ] 披露的利益冲突

### 开放科学
- [ ] 计划数据共享（符合道德标准）
- [ ] 分析代码共享
- [ ] 可用材料
- [ ] 预印本已发布
- [ ] 尽可能开放获取出版物

## 学习后注意事项

### 数据分析
- [ ] 遵循预先注册的计划
- [ ] 清楚地标记偏差和探索性分析
- [ ] 检查假设
- [ ] 报告所有结果
- [ ] 报告效应量和 CI，而不仅仅是 p 值

### 解释
- [ ] 有数据支持的结论
- [ ] 承认限制
- [ ] 考虑的替代解释
- [ ] 讨论了普遍性
- [ ] 解决了临床/实际意义

### 传播
- [ ] 无论结果如何都发布（减少发布偏差）
- [ ] 出席会议
- [ ] 与参与者分享发现（在适当的情况下）
- [ ] 与相关利益相关者沟通
- [ ] 简单语言摘要

### 后续步骤
- [ ] 需要复制吗？
- [ ] 确定后续研究
- [ ] 计划进行机制研究
- [ ] 考虑临床应用

## 要避免的常见陷阱

- [ ] 无功效分析 → 动力不足的研究
- [ ] 看到数据后形成的假设（HARKing）
- [ ] 可行时不致盲 → 偏差
- [ ] P-hacking（数据钓鱼，可选停止）
- [ ] 未经校正的多次测试 → 误报
- [ ] 对照组不充分
- [ ] 未解决混杂问题
- [ ] 仪器未经验证
- [ ] 高流失率未得到解决
- [ ] 择优结果报告
- [ ] 相关数据的因果语言
- [ ] 忽略统计检验的假设
- [ ] 不预注册会改变文献偏见
- [ ] 未披露的利益冲突

- [ ] 假设是可检验的且具体的
- [ ] 研究设计适当

- [ ] 混淆得到控制
- [ ] 实施随机化和盲法

- [ ] 协议已记录
- [ ] 备份计划存在问题

## 记住

**好的实验设计是：**
- 提出明确的问题
- 最大限度地减少偏见
- 最大化有效性
- 适当的推论
- 透明度
- 再现性

**思考这些问题的最佳时间是在收集数据之前，而不是之后。**
