# 科学批判性思维

## 概述

批判性思维是评估科学严谨性的系统过程。使用 GRADE 和 Cochrane ROB 框架评估方法、实验设计、统计有效性、偏差、混杂因素和证据质量。运用这项技能对科学主张进行批判性分析。

## 何时使用此技能

该技能应该在以下情况下使用：
- 评估研究方法和实验设计
- 评估统计有效性和证据质量
- 识别研究中的偏见和混杂因素
- 审查科学主张和结论
- 进行系统评价或荟萃分析
- 应用 GRADE 或 Cochrane 偏倚风险评估
- 提供研究论文的批判性分析

## 核心能力

### 1.方法论批判

评估研究方法的严谨性、有效性和潜在缺陷。

**申请时间：**
- 审查研究论文
- 评估实验设计
- 评估研究方案
- 规划新的研究

**评估框架：**

1. **研究设计评估**
   - 设计是否适合研究问题？
   - 设计能否支持所提出的因果关系主张？
   - 比较组是否适当且充分？
   - 考虑实验、准实验或观察设计是否合理

2. **有效性分析**
   - **内部有效性：** 我们可以相信因果推论吗？
     - 检查随机化质量
     - 评估混杂控制
     - 评估选择偏差
     - 审查流失/辍学模式
   - **外部有效性：** 结果是否具有普遍性？
     - 评估样本代表性
     - 考虑环境的生态有效性
     - 评估条件是否符合目标应用
   - **构造有效性：** 措施是否捕获了预期的构造？
     - 审查测量验证
     - 检查操作定义
     - 评估措施是直接的还是代理的
   - **统计结论的有效性：** 统计推论是否合理？
     - 验证足够的功率/样本量
     - 检查假设合规性
     - 评估测试的适当性

3. **控制和致盲**
   - 随机化是否正确实施（序列生成、分配隐藏）？
   - 盲法是否可行并已实施（参与者、提供者、评估者）？
   - 控制条件是否适当（安慰剂、主动对照、不治疗）？
   - 性能或检测偏差会影响结果吗？

4. **测量质量**
   - 仪器是否经过验证且可靠？
   - 措施是尽可能客观，还是带有公认局限性的主观措施？
   - 结果评估是否标准化？
   - 是否使用多种措施来对调查结果进行三角测量？

**参考：** 详细原理请参阅`references/scientific_method.md`，全面设计清单请参阅`references/experimental_design.md`。

### 2. 偏差检测

识别并评估可能扭曲研究结果的潜在偏见来源。

**申请时间：**
- 审查已发表的研究
- 设计新的研究
- 解释相互矛盾的证据
- 评估研究质量

**系统偏差审查：**

1. **认知偏差（研究员）**
   - **确认偏差：** 是否仅强调支持性发现？
   - **HARKing：** 假设是先验提出的还是在看到结果后形成的？
   - **发表偏倚：** 文献中是否缺少阴性结果？
   - **精挑细选：** 证据是否有选择性地报告？
   - 检查预注册和分析计划的透明度

2. **选择偏差**
   - **抽样偏差：** 样本是否代表目标人群？
   - **志愿者偏见：** 参与者是否以系统的方式进行自我选择？
   - **自然流失偏差：** 各组之间的辍学率是否存在差异？
   - **幸存者偏差：** 样本中是否只有“幸存者”可见？
   - 检查参与者流程图并比较基线特征

3. **测量偏差**
   - **观察者偏差：** 期望会影响观察结果吗？
   - **回忆偏差：** 回顾性报告是否系统上不准确？
   - **社会期望：** 回应是否偏向于可接受性？
   - **仪器偏差：** 测量工具是否会系统性地出错？
- 评估盲法、验证和测量客观性

4. **分析偏差**
   - **P-hacking：** 是否进行了多次分析直至出现显着性？
   - **结果转换：** 非显着结果是否被显着结果取代？
   - **选择性报告：** 是否报告了所有计划的分析？
   - **分组捕捞：** 分组分析是否未经修正进行？
   - 检查研究注册并与已发布的结果进行比较

5. **混杂**
   - 哪些变量可能影响暴露和结果？
   - 是否测量和控制了混杂因素（统计或设计）？
   - 无法测量的混杂因素可以解释研究结果吗？
   - 有其他合理的解释吗？

**参考：** 请参阅 `references/common_biases.md` 了解带有检测和缓解策略的综合偏差分类法。

### 3.统计分析评估

严格评估统计方法、解释和报告。

**申请时间：**
- 回顾定量研究
- 评估数据驱动的索赔
- 评估临床试验结果
- 回顾荟萃分析

**统计审查清单：**

1. **样本大小和功效**
   - 是否进行了先验功率分析？
   - 样本是否足以检测有意义的影响？
   - 研究是否动力不足（常见问题）？
   - 小样本的显着结果是否会导致效应量夸大？

2. **统计测试**
   - 测试是否适合数据类型和分布？
   - 测试假设是否经过检查并得到满足？
   - 参数测试是否合理，或者是否应该使用非参数替代方案？
   - 分析是否与研究设计相匹配（例如，配对与独立）？

3. **多重比较**
   - 是否测试了多个假设？
   - 是否应用了修正（Bonferroni、FDR、其他）？
   - 主要结果与次要/探索性结果有区别吗？
   - 多次测试的结果可能是误报吗？

4. **P 值解释**
   - p 值是否正确解释（如果 null 为 true，则数据的概率）？
   - 无意义是否被错误地解释为“无影响”？
   - 统计显着性是否与实际重要性相结合？
   - 是否报告了精确的 p 值，还是仅“p < .05”？
   - 0.05 以下是否存在可疑聚类？

5. **效果大小和置信区间**
   - 效应大小是否与显着性一起报告？
   - 是否提供置信区间来显示精度？
   - 效应大小有实际意义吗？
   - 标准化效应大小是否用特定领域的背景来解释？

6. **缺失数据**
   - 丢失了多少数据？
   - 是否考虑缺失数据机制（MCAR、MAR、MNAR）？
   - 如何处理缺失数据（删除、插补、最大似然）？
   - 缺失数据会导致偏差吗？

7. **回归和建模**
   - 模型是否过度拟合（预测变量太多，没有交叉验证）？
   - 预测是否超出数据范围（外推法）？
   - 多重共线性问题是否得到解决？
   - 是否检查了模型假设？

8. **常见陷阱**
   - 相关性被视为因果关系
   - 忽略均值回归
   - 忽视基本利率
   - 德克萨斯神枪手谬误（在噪音中寻找模式）
   - 辛普森悖论（子群混淆）

**参考：**参见`references/statistical_pitfalls.md`了解详细的陷阱和正确做法。

### 4. 证据质量评估

系统地评估证据的强度和质量。

**申请时间：**
- 权衡决策证据
- 进行文献综述
- 比较相互矛盾的发现
- 确定结论的可信度

**证据评估框架：**

1. **研究设计层次结构**
   - 系统评价/荟萃分析（干预效果最高）
   - 随机对照试验
   - 队列研究
   - 病例对照研究
   - 横断面研究
   - 案例系列/报告
   - 专家意见（最低）

   **重要提示：** 更高水平的设计并不总是更好的质量。设计良好的观察性研究可能比实施不佳的随机对照试验更有力。

2. **设计类型中的质量**
   - 偏倚风险评估（使用适当的工具：Cochrane ROB、Newcastle-Ottawa 等）
   - 方法论的严谨性
   - 透明度和报告完整性
   - 利益冲突

3. **等级考虑因素（如果适用）**
   - 从设计类型开始（RCT = 高，观察 = 低）
   - **降级为：**
     - 偏见风险
     - 研究之间的不一致
- 间接性（错误的群体/干预/结果）
     - 不精确（宽置信区间、小样本）
     - 发表偏见
   - **升级为：**
     - 效应量大
     - 剂量-反应关系
     - 混杂因素会减少（而不是增加）效果

4. **证据趋同**
   - **在以下情况下更强：**
     - 多次独立复制
     - 不同的研究小组和环境
     - 不同的方法论得出相同的结论
     - 机制和经验证据一致
   - **当以下情况时较弱：**
     - 单一研究或研究小组
     - 文献中相互矛盾的发现
     - 出版偏见明显
     - 没有复制尝试

5. **背景因素**
   - 生物学/理论合理性
   - 与既定知识的一致性
   - 时间性（原因先于结果）
   - 关系的特殊性
   - 协会实力

**参考：** 请参阅`references/evidence_hierarchy.md` 了解详细的层次结构、GRADE 系统和质量评估工具。

### 5.逻辑谬误识别

检测并指出科学论证和主张中的逻辑错误。

**申请时间：**
- 评估科学主张
- 审查讨论/结论部分
- 评估科普传播
- 识别有缺陷的推理

**科学中常见的谬误：**

1. **因果谬误**
   - **Post hoc ergo propter hoc：**“B 跟随 A，所以 A 导致 B”
   - **相关 = 因果关系：** 与因果关系的令人困惑的关联
   - **反向因果关系：** 误认为原因为结果
   - **单一原因谬误：** 将复杂的结果归因于一个因素

2. **泛化谬误**
   - **仓促概括：** 小样本得出广泛结论
   - **轶事谬误：** 个人故事作为证据
   - **精挑细选：** 仅选择支持证据
   - **生态谬误：** 群体模式应用于个人

3. **权威和来源谬误**
   - **诉诸权威：**“专家说了，所以是真的”（无证据）
   - **人身攻击：** 攻击人，而不是争论
   - **遗传谬误：** 以出身而非优劣来判断
   - **诉诸自然：**“自然=良好/安全”

4. **统计谬误**
   - **基本率忽略：** 忽略先验概率
   - **德克萨斯神枪手：** 在随机数据中寻找模式
   - **多重比较：** 不纠正多重测试
   - **检察官的谬论：** 将 P(E|H) 与 P(H|E) 混淆

5. **结构性谬误**
   - **错误的二分法：** 当存在更多选项时，“A 或 B”
   - **移动球门柱：** 达到证据标准后改变它们
   - **提出问题：** 循环推理
   - **稻草人：**歪曲论据来攻击他们

6. **特定科学的谬误**
   - **伽利略开局：**“他们嘲笑伽利略，所以我的边缘想法是正确的”
   - **无知的论证：**“没有被证明是错误的，所以是正确的”
   - **涅盘谬误：** 拒绝不完美的解决方案
   - **不可证伪性：**做出无法检验的主张

**识别谬误时：**
- 指出具体的谬误
- 解释为什么推理有缺陷
- 确定有效推论需要哪些证据
- 请注意，错误的推理并不能证明结论是错误的——只是这个论证不支持它

**参考：** 请参阅`references/logical_fallacies.md`，了解包含示例和检测策略的综合谬误目录。

### 6. 研究设计指导

为规划严谨的研究提供建设性指导。

**申请时间：**
- 帮助设计新实验
- 规划研究项目
- 审查研究计划
- 改进研究方案

**设计流程：**

1. **研究问题细化**
   - 确保问题具体、可回答且可证伪
   - 验证它解决了文献中的空白或矛盾
   - 确认可行性（资源、道德、时间）
   - 可操作地定义变量

2. **设计选型**
   - 将设计与问题相匹配（因果→实验；关联→观察）
   - 考虑可行性和道德约束
   - 选择科目间、科目内或混合设计
   - 如果测试多个因素，则规划因子设计

3. **偏差最小化策略**
   - 尽可能实施随机化
   - 在所有可行级别（参与者、提供者、评估者）计划盲法
   - 识别并计划控制混杂因素（随机化、匹配、分层、统计调整）
   - 标准化所有程序
- 计划尽量减少人员流失

4. **样品规划**
   - 进行先验功效分析（指定预期效果、所需功效、阿尔法）
   - 考虑样本量的流失
   - 定义明确的纳入/排除标准
   - 考虑招聘策略和可行性
   - 样本代表性计划

5. **测量策略**
   - 选择经过验证、可靠的仪器
   - 尽可能使用客观措施
   - 规划关键结构的多项措施（三角测量）
   - 确保措施对预期变化敏感
   - 建立评估者间的可靠性程序

6. **分析计划**
   - 预先指定所有假设和分析
   - 明确指定主要结果
   - 通过假设检查计划统计测试
   - 指定如何处理丢失的数据
   - 计划报告效应大小和置信区间
   - 考虑多重比较修正

7. **透明度和严谨性**
   - 预先注册研究和分析计划
   - 使用报告指南（CONSORT、STROBE、PRISMA）
   - 计划报告所有结果，而不仅仅是重要结果
   - 区分验证性分析和探索性分析
   - 致力于数据/代码共享

**参考：** 请参阅`references/experimental_design.md`，了解涵盖从问题到传播的所有阶段的综合设计清单。

### 7. 索赔评估

系统地评估科学主张的有效性和支持性。

**申请时间：**
- 评估论文中的结论
- 评估研究的媒体报道
- 审查摘要或简介声明
- 检查数据是否支持结论

**索赔评估流程：**

1. **确定索赔**
   - 到底主张什么？
   - 是因果断言、关联断言还是描述性断言？
   - 该主张有多强（已证实、有可能、建议、可能）？

2. **评估证据**
   - 提供什么证据？
   - 证据是直接的还是间接的？
   - 证据是否足以证明主张的强度？
   - 是否排除了其他解释？

3. **检查逻辑连接**
   - 从数据中得出结论吗？
   - 是否有逻辑上的跳跃？
   - 相关数据是否用于支持因果关系？
   - 是否承认限制？

4. **评估比例**
   - 置信度与证据强度成正比吗？
   - 回避词的使用是否恰当？
   - 限制是否被淡化？
   - 猜测是否有明确标签？

5. **检查是否过度概括**
   - 主张是否超出了研究样本范围？
   - 是否承认人口限制？
   - 是否认识到上下文相关性？
   - 是否包含有关概括的警告？

6. **危险信号**
   - 相关研究的因果语言
   - “证明”或绝对确定性
   - 精挑细选的引文
   - 忽略矛盾的证据
   - 消除限制
   - 超越数据的推断

**提供具体反馈：**
- 引用有问题的主张
- 解释需要什么证据来支持它
- 如果有必要，建议适当的对冲语言
- 区分数据（发现了什么）和解释（它意味着什么）

## 申请指南

### 一般方法

1. **要有建设性**
   - 识别优势和劣势
   - 提出改进建议而不是仅仅批评
   - 区分致命缺陷和次要缺陷
   - 认识到所有研究都有局限性

2. **具体一点**
   - 指向特定实例（例如，“表 2 显示...”或“在方法部分...”）
   - 引用有问题的陈述
   - 提供问题的具体例子
   - 提及违反的具体原则或标准

3. **保持比例**
   - 将批评的严重程度与问题的重要性相匹配
   - 区分对有效性的主要威胁和次要问题
   - 考虑问题是否影响主要结论
   - 承认自己评估中的不确定性

4. **应用一致的标准**
   - 在所有研究中使用相同的标准
   - 不要对你不喜欢的发现采用更严格的标准
   - 承认你自己的潜在偏见
   - 基于方法而非结果进行判断

5. **考虑背景**
   - 承认实际和道德约束
   - 考虑效应大小和方法的特定领域规范
   - 识别探索性与确认性背景
   - 在评估研究时考虑资源限制

### 提出批评时

**结构反馈为：**

1. **摘要：** 评估内容的简要概述
2. **优势：** 哪些方面做得好（对于可信度和学习很重要）
3. **关注点：** 按严重程度排列的问题
   - 关键问题（威胁主要结论的有效性）
   - 重要问题（影响解释但不致命）
   - 小问题（值得注意但不改变结论）
4. **具体建议：** 可操作的改进建议
5. **总体评估：** 关于证据质量和可以得出的结论的平衡结论

**使用精确的术语：**
- 指出具体的偏见、谬误和方法论问题
- 参考既定标准和指南
- 引用科学方法论的原理
- 准确使用技术术语

### 当不确定时

- **承认不确定性：**“这可能是 X 或 Y；需要的附加信息是 Z”
- **提出澄清问题：**“[方法细节]完成了吗？这会影响解释。”
- **提供条件评估：**“如果 X 完成，则 Y 完成；如果没有，则 Z 值得关注”
- **注意哪些附加信息可以解决不确定性**

## 记住

**科学批判性思维是关于：**
- 使用既定原则进行系统评估
- 提高科学水平的建设性批评
- 置信度与证据强度成比例
- 不确定性和局限性的透明度
- 标准的一致应用
- 认识到所有研究都有局限性
- 怀疑与开放证据之间的平衡

**始终区分：**
- 数据（观察到的内容）和解释（意味着什么）
- 相关性和因果关系
- 统计意义和实际意义
- 探索性和验证性发现
- 什么是已知的，什么是不确定的
- 反对索赔的证据和无效的证据

**批判性思维的目标：**
1. 准确识别优势和劣势
2. 确定支持哪些结论
3. 认识到局限性和不确定性
4.对今后的工作提出改进建议
5. 增进科学理解
