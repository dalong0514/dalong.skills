# 困惑度模型比较

有关通过 OpenRouter 提供的不同 Perplexity 模型以及何时使用每个模型的指南。

## 可用型号

所有 Perplexity 模型均通过 OpenRouter 使用以下格式访问：
`openrouter/perplexity/[model-name]`

### 声纳专业搜索

**型号 ID**：`openrouter/perplexity/sonar-pro-search`

**最适合：**
- 复杂的多步骤研究问题
- 需要深入分析和综合的查询
- 需要综合探源的情况
- 跨多个领域的比较分析
- 需要代理推理工作流程的研究

**特点：**
- 最先进的代理搜索系统
- 执行多步骤推理工作流程
- 使用工具和中间查询
- 提供最全面的答案
- 大量加工导致成本较高

**使用案例：**
-“对竞争性 CAR-T 细胞治疗方法进行全面分析，包括机制差异、临床结果和成本效益”
-“跨多个指标比较药物发现的量子计算方法与传统计算方法”
- 需要从多个来源进行综合的研究问题

**定价**（大约）：
- 输入：3 美元/百万代币
- 产出：15 美元/百万代币
- 请求费：每 1000 个请求 18 美元

**上下文窗口**：200K 代币

### 声纳专业版

**型号 ID**：`openrouter/perplexity/sonar-pro`

**最适合：**
- 通用研究和搜索
- 平衡的性能和成本
- 标准科学查询
- 快速信息收集
- 大多数用例

**特点：**
- 增强了基础声纳的功能
- 质量和成本的良好平衡
- 对于大多数查询来说都是可靠的
- 比专业搜索更快
- 推荐的默认选择

**使用案例：**
- “CRISPR碱基编辑的最新进展是什么？”
- “总结最近阿尔茨海默病治疗的临床试验”
- “解释变压器架构如何在现代法学硕士中工作”
- 标准文献检索
- 技术文档查询

**定价**（大约）：
- 比专业搜索成本更低
- 良好的性价比

**上下文窗口**：200K 代币

### 声纳

**最适合：**

- 简单的事实调查
- 大容量查询
- 快速查找

**特点：**
- 具有稳定性能的基础模型
- 最具成本效益的选择
- 更快的响应时间
- 适合简单查询
- 精度低于 Pro 版本

**使用案例：**

- 基本信息检索

**定价**（大约）：
- 成本最低的选择
- 最适合大容量简单查询

**上下文窗口**：200K 代币

## 选型指南

### 决策树

```
Is your query complex and requiring deep multi-step analysis?
├─ YES → Use Sonar Pro Search
└─ NO → Continue

Does your query require explicit step-by-step reasoning?
├─ YES → Use Sonar Reasoning Pro (complex) or Sonar Reasoning (simple)
└─ NO → Continue

Is this a standard research or information query?
├─ YES → Use Sonar Pro (recommended default)
└─ NO → Continue

Is this a simple fact-finding or basic lookup?
├─ YES → Use Sonar (cost-effective)
└─ NO → Use Sonar Pro (safe default)
```

- 由于明确的思维步骤，推理模型速度较慢

## 特别注意事项

### 上下文窗口

所有型号均支持 200K 令牌上下文窗口：
- 足以满足大多数查询
- 可以处理长文档或多个来源
- 考虑对非常大的分析进行分块

### 温度设置

不同的型号受益于不同的温度设置：

**声纳专业搜索：**
- 默认值：0.2（更专注、更分析）
- 对事实查询使用较低值 (0.0-0.1)
- 使用更高的（0.3-0.5）进行创意合成

**声纳推理专业版：**
- 默认值：0.2
- 保持低电平 (0.0-0.2) 以保持逻辑一致性
- 高温下推理质量会下降

**声纳专业版/声纳：**
- 默认值：0.2
- 根据查询类型进行调整（事实性与探索性）

### 速率限制和配额

OpenRouter 强制执行速率限制：
- 检查您的 OpenRouter 仪表板的当前限制
- 考虑对大批量使用进行请求批处理
- 使用 OpenRouter 的跟踪工具监控成本

### API 密钥安全

**最佳实践：**
- 切勿将 API 密钥提交给版本控制
- 使用环境变量或 .env 文件
- 定期轮换钥匙
- 监控意外活动的使用情况
- 对不同的项目使用单独的密钥

## 总结

**默认建议**：对于大多数科学查询，从 **Sonar Pro** 开始。

**何时升级：**
- 复杂的多步骤分析 → Sonar Pro 搜索
- 需要显式推理 → Sonar Reasoning Pro

**何时降级：**
- 简单的事实或查找 → 声纳
- 对成本敏感的批量查询 → 声纳

**记住**：最佳模型取决于您的具体用例、预算和质量要求。监控您的使用情况并根据结果调整型号选择。
