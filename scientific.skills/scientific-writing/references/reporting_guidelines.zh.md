# 科学研究报告指南

## 概述

报告指南是基于证据的建议，建议在报告特定类型的研究时应包含哪些信息。他们提供清单和流程图，以确保报告完整、准确和透明，这对于读者评估研究的有效性以及其他研究人员复制研究工作至关重要。

EQUATOR 网络（提高健康研究的质量和透明度）维护着一个全面的报告指南库。使用适当的报告指南可以提高稿件质量并增加出版物接受的可能性。

## 为什么要使用报告指南？

### 好处

**对于作者：**
- 确保没有忘记任何重要的事情
- 提高接受率
- 改进稿件组织
- 减少审稿人对附加信息的请求

**对于读者和审稿人：**
- 能够对研究有效性进行严格评估
- 促进系统评价和荟萃分析
- 提高对实际做了什么的理解

**对于科学：**
- 提高再现性
- 减少研究浪费
- 提高透明度
- 实现更好的证据合成

### 何时使用

- **在研究设计期间**：许多指南包括方案版本（例如试验方案的 SPIRIT）
- **在稿件起草过程中**：使用清单确保涵盖所有项目
- **提交之前**：验证遵守情况并经常随稿件一起提交清单
- **许多期刊要求**：报告指南清单作为提交的一部分

## 按研究类型划分的主要报告指南

### CONSORT - 随机对照试验

**全名：** 报告试验综合标准

**何时使用：** 任何随机对照试验 (RCT)，包括试点和可行性试验

**最新版本：** CONSORT 2010（更新声明）

**关键部件：**
- **清单**：25 个项目，涵盖标题、摘要、简介、方法、结果、讨论
- **流程图**：参与者的注册、分配、跟进和分析流程

**主要清单项目：**
1. 标题将研究确定为随机试验
2. 结构化摘要
三、科学背景和原理
4. 具体目标和假设
5. 试验设计描述（并行、交叉、阶乘等）
六、参赛者资格标准
7. 数据收集的设置和地点
8. 足够详细地描述干预措施以供复制
9. 主要和次要结果的定义
10. 样本量确定和功效计算
11. 随机化序列生成
12.分配隐藏机制
13. 盲法实施
14.统计方法
15. 参与者流量及退出原因
16. 招募日期和后续日期
17. 基线特征表
18. 每个结果的分析结果
19. 危害和不良事件
20. 试验限制
21. 普遍性
22. 解释与结果一致
23. 试用注册号
24. 完整协议访问
25. 资金来源

**特定设计的扩展：**
- 用于整群随机试验的 CONSORT
- 非劣效性和等效性试验的 CONSORT
- CONSORT 实用试验
- CONSORT 用于交叉试验
- CONSORT 用于 N-of-1 试验
- CONSORT 用于阶梯式楔块设计

**访问地点：** http://www.consort-statement.org/

### STROBE - 观察研究

**全名：** 加强流行病学观察研究报告

**何时使用：** 队列研究、病例对照研究和横断面研究

**最新版本：** STROBE 2007（广泛采用的标准）

**涵盖的关键研究设计：**
- **群组**：及时跟踪暴露和未暴露的群体
- **病例对照**：比较病例和对照之间的暴露史
- **横截面**：同时测量暴露和结果

**主要清单项目（22 项）：**
1. 标题和摘要表明研究设计
2. 背景和理由
3. 目标
4. 研究设计的基本原理
5. 场景、地点和日期
六、资格标准及评选办法
7. 明确定义的变量（结果、暴露、混杂因素）
八、数据来源及测量方法
9. 偏见管理策略
10. 研究规模的合理性
11. 定量变量的处理
12. 统计方法，包括混杂和交互作用
13. 敏感性分析
14. 参加者流量及未参加原因
15. 描述性数据，包括随访时间
16. 结果数据
17. 未经调整和调整估计的主要结果
18. 其他分析（亚组、敏感性分析）
19. 主要结果总结
20. 潜在偏见讨论的局限性
21. 解释和概括
22. 资金来源和作用

**扩展：**
- STROBE-ME（分子流行病学）
- 记录（定期收集的健康数据）
- STROBE-RDS（受访者驱动抽样）

**访问位置：** https://www.strobe-statement.org/

### PRISMA - 系统评价和荟萃分析

**全名：**系统评价和荟萃分析的首选报告项目

**何时使用：** 带或不带荟萃分析的系统评价

**最新版本：** PRISMA 2020（重大更新）

**关键部件：**
- **清单**：27 项涵盖所有部分
- **流程图**：研究选择过程

**主要部分：**
1. **标题**：确定为系统评价/荟萃分析
2. **摘要**：结构化摘要
3. **简介**：基本原理和目标
4. **方法**：
   - 资格标准
   - 信息来源（数据库、日期）
   - 搜索策略（至少一个数据库的完整策略）
   - 选择过程
   - 数据收集过程
   - 提取的数据项
   - 偏见风险评估
   - 效果措施
   - 合成方法
   - 报告偏见评估
   - 确定性评估（例如，GRADE）
5. **结果**：
   - 研究选择流程图
   - 研究特点
   - 评估结果存在偏差的风险
   - 合成结果（荟萃分析，如果适用）
   - 报告偏见
   - 证据的确定性
6. **讨论**：
   - 限制
   - 口译
   - 影响

**扩展：**
- PRISMA 摘要
- PRISMA 协议 (PRISMA-P)
- 用于网络元分析的 PRISMA
- PRISMA 用于范围界定审查 (PRISMA-ScR)
- PRISMA 用于个体患者数据
- PRISMA 提高诊断测试准确性
- PRISMA 用于以股票为重点的审查

**访问位置：** http://www.prisma-statement.org/

## 如何使用报告指南

### 学习计划期间

1. **根据研究设计确定相关指南**
2. **审查需要规划的清单项目**（例如随机化、盲法）
3. **设计研究**以确保捕获所有必需的元素
4. **考虑方案指南**（例如，用于试验的 SPIRIT）

### 稿件起草期间

1. **从指南网站下载清单**
2. **系统地完成每个项目**
3. **注意手稿中每个项目的处理位置**（页码/行号）
4. **修改手稿**以包括缺失的项目
5. **酌情使用流程图**

### 提交前

1. **完整的正式清单**及页码
2. **检查所有项目**是否已得到充分解决
3. **如果期刊需要，请在提交时附上清单**
4. **在求职信或方法中注意遵守指南**

### 清单条目示例

```
Item 7: Eligibility criteria for participants, and the settings and locations where the data were collected

Recruitment occurred from three memory clinics in Boston, MA, between January 2022 and December 2023."
```

## 常见错误以及如何避免它们

### 错误 1：根本不使用指南

**影响：** 缺少关键信息，接受机会较低

**解决方案：** 从研究计划阶段确定并使用适当的指南

### 错误 2：仅在稿件完成后才使用指南

**影响：** 可能意识到未收集或记录关键数据

**解决方案：** 在研究设计和数据收集期间审查指南

### 错误 3：清单完成不完整

**影响：** 遗漏的物品仍未报告

**解决方案：** 系统地处理每个清单项目

### 错误 4：使用过时的指南

**影响：** 缺少报告标准的最新改进

**解决方案：** 始终在官方指南网站上检查最新版本

### 错误 5：使用错误的研究设计指南

**影响：** 未报告重要的设计特定元素

**解决方案：** 仔细地将研究设计与适当的指南相匹配

### 错误 6：未在需要时提交清单

**影响：** 编辑部拒绝或延迟

**解决方案：** 检查期刊提交指南并包括清单

### 错误 7：笼统报告而不具体

**影响：** 复制或评估的细节不足

**解决方案：** 为每个项目提供具体、详细的信息

## 特定场地的报告要求

### 按场地类型划分的报告标准

|场地类型 |使用指南 |透明度要求|
|------------|--------------|----------------------------|
| **医学期刊** |强制（CONSORT、STROBE 等）|提交时需要核对清单 |
| **PLOS/BMC** |学习类型强制 |作为补充上传的清单 |
| **自然/科学** |推荐|强调方法完整性|
| **机器学习会议** |没有正式的指导方针|所需的再现性详细信息 |

### ML 会议报告标准

**NeurIPS/ICML/ICLR 重现性要求：**
- **数据集**：名称、版本、访问方法、预处理
- **代码**：可用性声明； GitHub 常见问题
- **超参数**：报告的所有设置（学习率、批量大小等）
- **种子**：随机种子以实现可重复性
- **计算资源**：使用的 GPU、训练时间
- **统计显着性**：误差线、置信区间、多次运行
- **更广泛的影响**声明 (NeurIPS)：社会影响

**要包括的内容（通常在附录中）：**
- 完整的超参数设置
- 培训细节和收敛标准
- 硬件规格
- 软件版本（PyTorch 2.0等）
- 数据集分割和任何预处理
- 评估指标和协议

### 执行和评估

**检查什么：**
- **医学期刊**：上传清单；方法中的遵守声明；系统完整性
- **PLOS/BMC**：某些设计的强制性清单；强调再现性
- **高影响力**：复制方法充足（通常不需要清单）
- **机器学习会议**：可重复性检查表（NeurIPS）；代码可用性的期望越来越高

**导致拒绝的常见问题：**
- 缺少必需的清单（医学期刊）
- 复制方法细节不足
- 缺少关键信息（随机化、致盲、功效计算）
- 需要时没有数据/代码可用性声明
