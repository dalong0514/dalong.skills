# gget 数据库信息

gget 模块查询的数据库概述，包括更新频率和重要注意事项。

## 重要提示

gget 查询的数据库不断更新，有时会改变其结构。 gget 模块每两周自动测试一次，并在必要时更新以匹配新的数据库结构。始终保持 gget 更新：

```bash
pip install --upgrade gget
```

## 数据库目录

### 基因组参考数据库

#### 合奏
- **使用者：** gget ref、gget search、gget info、gget seq
- **描述：** 全面的基因组数据库，带有脊椎动物和无脊椎动物物种的注释
- **更新频率：** 定期发布（编号）；大约每 3 个月发布一次新产品
- **访问：** FTP 下载、REST API
- **网站：** https://www.ensembl.org/
- **注释：**
  - 支持脊椎动物和无脊椎动物基因组
  - 可以指定版本号以实现可重复性
  - 适用于常见物种的快捷方式（“人类”、“小鼠”）

#### UCSC 基因组浏览器
- **使用者：** gget blat
- **描述：** 带有 BLAT 比对工具的基因组浏览器数据库
- **更新频率：**定期更新新组件
- **访问：** Web 服务 API
- **网站：** https://genome.ucsc.edu/
- **注释：**
  - 多个基因组可用（hg38、mm39等）
  - 针对脊椎动物基因组优化的 BLAT

### 蛋白质和结构数据库

#### UniProt
- **使用者：** gget info、gget seq（氨基酸序列）、gget elm
- **描述：** 通用蛋白质资源，全面的蛋白质序列和功能信息
- **更新频率：** 定期发布（Swiss-Prot 每周发布，TrEMBL 每月发布）
- **访问：** REST API
- **网站：** https://www.uniprot.org/
- **注释：**
  - Swiss-Prot：手动注释和审核
  - TrEMBL：自动注释

#### NCBI（国家生物技术信息中心）
- **使用者：** gget info、gget bgee（对于非 Ensembl 物种）
- **描述：** 具有广泛交叉引用的基因和蛋白质数据库
- **更新频率：** 持续更新
- **访问：** 电子实用程序 API
- **网站：** https://www.ncbi.nlm.nih.gov/
- **数据库：** 基因、蛋白质、RefSeq

#### RCSB PDB（蛋白质数据库）
- **使用者：** gget pdb
- **描述：** 蛋白质和核酸的 3D 结构数据存储库
- **更新频率：** 每周更新
- **访问：** REST API
- **网站：** https://www.rcsb.org/
- **注释：**
  - 实验确定的结构（X 射线、NMR、冷冻电镜）
  - 包括有关实验和出版物的元数据

#### ELM（真核线性基序）
- **使用者：** gget elm
- **描述：** 真核蛋白质功能位点数据库
- **更新频率：** 定期更新
- **访问：** 下载的数据库（通过 gget setup elm）
- **网站：** http://elm.eu.org/
- **注释：**
  - 首次使用前需要本地下载
  - 包含经过验证的图案和图案

### 序列相似性数据库

#### BLAST 数据库 (NCBI)
- **使用者：** ggetblast
- **描述：** 用于 BLAST 搜索的预格式化数据库
- **更新频率：** 定期更新
- **访问：** NCBI BLAST API
- **数据库：**
  - **核苷酸：** nt（所有 GenBank）、refseq_rna、pdbnt
  - **蛋白质：** nr（非冗余）、swissprot、pdbaa、refseq_ Protein
- **注释：**
  - nt 和 nr 是非常大的数据库
  - 考虑专门的数据库以实现更快、更有针对性的搜索

### 表达和相关数据库

#### 拱门4
- **使用者：** gget archs4
- **描述：** 公开可用的 RNA-seq 数据的大规模挖掘
- **更新频率：** 定期更新新样本
- **访问：** HTTP API
- **网站：** https://maayanlab.cloud/archs4/
- **数据：**
  - 人类和小鼠 RNA-seq 数据
  - 相关矩阵
  - 组织表达图谱
- **引用：** Lachmann 等人，《自然通讯》，2018

#### CZ CELLxGENE 发现
- **使用者：** gget cellxgene
- **描述：** 来自多项研究的单细胞 RNA-seq 数据
- **更新频率：** 不断添加新数据集
- **访问：** Census API（通过 cellxgene-census 包）
- **网站：** https://cellxgene.cziscience.com/
- **数据：**
  - 单细胞 RNA-seq 计数矩阵
  - 细胞类型注释
  - 组织和疾病元数据
- **注释：**
  - 需要 gget 设置 cellxgene
  - 基因符号区分大小写
  - 可能不支持最新的Python版本

#### 哎呀
- **使用者：** gget bgee
- **描述：** 基因表达和直系数据库
- **更新频率：** 定期发布
- **访问：** REST API
- **网站：** https://www.bgee.org/
- **数据：**
- 跨组织和发育阶段的基因表达
  - 跨物种的直系关系
- **引用：** Bastian 等人，2021

### 功能和通路数据库

#### 丰富 / modEnrichr
- **使用者：** gget richr
- **描述：** 基因集富集分析网络服务
- **更新频率：**定期更新底层数据库
- **访问：** REST API
- **网站：** https://maayanlab.cloud/Enrichr/
- **包括的数据库：**
  - KEGG途径
  - 基因本体论（GO）
  - 转录因子目标 (ChEA)
  - 疾病协会（GWAS 目录）
  - 细胞类型标记（PanglaoDB）
- **注释：**
  - 支持多种模式生物
  - 可以提供背景基因列表以进行定制富集

### 疾病和药物数据库

#### 开放目标
- **使用者：** gget opentargets
- **描述：** 疾病目标关联的综合平台
- **更新频率：** 定期发布（每季度）
- **访问：** GraphQL API
- **网站：** https://www.opentargets.org/
- **数据：**
  - 疾病协会
  - 药物信息和临床试验
  - 目标易处理性
  - 药物遗传学
  - 基因表达
  - DepMap 基因疾病影响
  - 蛋白质-蛋白质相互作用

#### cBioPortal
- **使用者：** gget cbio
- **描述：** 癌症基因组学数据门户
- **更新频率：** 不断添加新研究
- **访问：** Web API，可下载数据集
- **网站：** https://www.cbioportal.org/
- **数据：**
  - 突变、拷贝数改变、结构变异
  - 基因表达
  - 临床数据
- **注释：**
  - 大型数据集；推荐缓存
  - 多种癌症类型和可用研究

#### COSMIC（癌症体细胞突变目录）
- **使用者：** gget cosmic
- **描述：** 综合癌症突变数据库
- **更新频率：** 定期发布
- **访问：**下载（需要帐户和许可证才能用于商业用途）
- **网站：** https://cancer.sanger.ac.uk/cosmic
- **数据：**
  - 癌症中的体细胞突变
  - 基因普查
  - 细胞系数据
  - 耐药突变
- **重要：**
  - 免费供学术使用
  - 商业用途需支付许可费
  - 需要 COSMIC 帐户凭据
  - 查询前必须下载数据库

### 人工智能和预测服务

#### AlphaFold2（DeepMind）
- **使用者：** gget alphafold
- **描述：** 用于蛋白质结构预测的深度学习模型
- **模型版本：** 本地执行的简化版本
- **访问：**本地计算（需要通过 gget 设置下载模型）
- **网站：** https://alphafold.ebi.ac.uk/
- **注释：**
  - 需要~4GB模型参数下载
  - 需要安装 OpenMM
  - 计算密集型
  - Python 版本特定要求

#### OpenAI API
- **使用者：** gget gpt
- **描述：**大型语言模型API
- **更新频率：** 定期发布新型号
- **访问：** REST API（需要 API 密钥）
- **网站：** https://openai.com/
- **注释：**
  - 默认型号：gpt-3.5-turbo
  - 免费套餐仅限帐户创建后 3 个月内
  - 设置计费限额以控制成本

### 处理数据库更新

1. **定期gget更新：**
   - 每两周更新一次 gget 以匹配数据库结构的变化
   - 检查发行说明中的重大更改

2. **错误处理：**
   - 数据库结构更改可能会导致暂时故障
   - 检查 GitHub 问题：https://github.com/pachterlab/gget/issues
   - 如果发生错误则更新 gget

3. **API速率限制：**
   - 实现大规模查询的延迟
   - 尽可能使用本地数据库（DIAMOND、COSMIC）
   - 缓存结果以避免重复查询

## 特定于数据库的最佳实践

### 合奏
- 使用物种快捷方式（“人类”、“小鼠”）以方便起见
- 指定版本号以确保可重复性
- 使用`gget ref --list_species`检查可用物种

### UniProt
- UniProt ID 比基因名称更稳定
- Swiss-Prot 注释是手动管理的并且更可靠
- 仅在需要时在 gget 信息中使用 PDB 标志（增加运行时间）

### 爆炸/爆炸
- 从默认参数开始，然后优化
- 使用专门的数据库（swissprot、refseq_ Protein）进行有针对性的搜索
- 考虑基于查询长度的 E 值截止

### 表达数据库
- CELLxGENE 中的基因符号区分大小写
- ARCHS4 相关数据基于共表达模式
- 解释结果时考虑组织特异性

### 癌症数据库
- cBioPortal：在本地缓存数据以进行重复分析
- COSMIC：根据您的需求下载适当的数据库子集
- 尊重商业用途的许可协议
